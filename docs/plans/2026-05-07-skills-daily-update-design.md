# Skills Daily Update - Design Document

## Overview

OpenCode SKILL for daily sync of openagent framework skills: detect updates from git repos, pack changed skills into zip, upload to Aliyun OSS.

## Directory Structure

```
skills-daily-update/
├── SKILL.md
├── scripts/
│   └── skills_update.py
├── config/
│   └── repos.json
└── state/
    └── last_commits.json
```

## CLI Design

```bash
python scripts/skills_update.py fetch [--framework openagent] [--lang en]
python scripts/skills_update.py check --repos config/repos.json --state state/last_commits.json
python scripts/skills_update.py pack --plan state/pack_plan.json [--output ./dist]
python scripts/skills_update.py upload --plan state/pack_plan.json
```

### Subcommands

| Command | Input | Output |
|---------|-------|--------|
| `fetch` | API endpoint | Skill list to stdout (JSON) |
| `check` | repos.json + last_commits.json | pack_plan.json |
| `pack` | pack_plan.json | zip files + updated pack_plan.json with zip_path |
| `upload` | pack_plan.json | uploaded zips + updated pack_plan.json with oss_url |

### Data Flow

```
repos.json ─┐
            ├─→ check ─→ pack_plan.json ─→ pack ─→ pack_plan.json(updated) ─→ upload
last_commits.json ─┘
```

## repos.json (LLM-maintained, initially empty)

```json
{
  "_default_repo": "https://gitee.com/zerone-agent/agent-use-skills.git",
  "_default_path": "awesome-skills/skills",
  "skills": {
    "skill-market": {
      "repo": null,
      "subdir": "skill-market"
    },
    "superpowers": {
      "repo": "https://github.com/xxx/superpowers.git",
      "subdir": null
    }
  }
}
```

- `repo: null` → use `_default_repo` + `_default_path` + `subdir`
- `repo` set → independent repo, `subdir` optional

## pack_plan.json (auto-generated by check)

```json
{
  "generated_at": "2026-05-07T10:30:00Z",
  "skills": [
    {
      "name": "skill-market",
      "source_path": "/tmp/skills-update/agent-use-skills/awesome-skills/skills/skill-market",
      "commit_hash": "abc1234",
      "zip_path": null,
      "oss_url": null
    }
  ]
}
```

- `check` fills: name, source_path, commit_hash
- `pack` fills: zip_path
- `upload` fills: oss_url

## OSS Upload Path

```
oss://{bucket}/skills/{date}/{skill-name}.zip
```

Example: `oss://my-bucket/skills/2026-05-07/skill-market.zip`

## Environment Variables

```
OSS_ACCESS_KEY_ID
OSS_ACCESS_KEY_SECRET
OSS_ENDPOINT
OSS_BUCKET
```

## Dependencies

- `requests` — API calls
- `oss2` — Aliyun OSS
- stdlib: `zipfile`, `json`, `argparse`, `subprocess`, `pathlib`, `tempfile`, `datetime`

## Update Detection

Git commit hash comparison:
1. `check` clones/pulls each repo
2. Gets current HEAD commit hash for each skill's directory
3. Compares with last_commits.json
4. Outputs only changed skills to pack_plan.json
5. After successful upload, LLM updates last_commits.json

## SKILL.md

- Trigger: "update skills", "daily update", "sync skills", "check skill updates"
- Checklist-driven execution for LLM
- LLM maintains repos.json (adds unknown skills, updates repo URLs)
- LLM can inspect intermediate outputs and adjust flow

## Core Classes

```python
class SkillsUpdater:
    fetch(framework, lang) -> list
    check(repos_path, state_path) -> str  # outputs pack_plan.json path
    pack(plan_path, output_dir) -> None
    upload(plan_path) -> None

class RepoManager:
    load_config(path) -> dict
    get_repo_url(skill_name) -> tuple[str, str]  # (repo_url, subdir)
    clone_or_pull(repo_url, dest) -> str  # returns commit hash

class StateStore:
    load(path) -> dict
    save(path, data) -> None
    get_commit(skill_name) -> str | None
```
